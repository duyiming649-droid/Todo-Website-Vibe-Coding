# Figma UI 设计 Prompt

以下 prompt 可直接复制到 Figma AI（或 Figma 的 Generate UI）中使用。按需选择桌面端或移动端版本。

---

## 一、主界面完整 Prompt（推荐一次生成）

```
设计一个待办事项管理应用的单页界面，桌面端，1440×900 画布。

【布局结构】
- 顶部：标题 "我的待办" + 统计面板（完成率 65%、待办 7 项）
- 中部左侧/上方：添加表单区域（标题输入框、描述文本框、分类下拉、优先级下拉、截止日期选择器、添加按钮）
- 中部：搜索框（占位符 "搜索标题或描述..."）+ 筛选栏（分类筛选、优先级筛选、状态筛选：全部/未完成/已完成）
- 主体：待办列表，每个待办卡片显示：
  - 左侧：完成状态勾选框
  - 标题、描述（若有）、分类标签、优先级标签（低/中/高用不同颜色区分）、截止日期
  - 右侧：编辑按钮、删除按钮

【设计风格】
- 简洁干净，不堆砌渐变和过多阴影
- 主色：单一主色（如深蓝或墨绿），辅以中性灰背景
- 字体：无衬线，有辨识度（如 Geist、Inter、思源黑体）
- 圆角：卡片和按钮统一使用 8px 或 12px
- 避免 emoji 和花哨图标
- 不要 AI 常见的紫色渐变或彩虹配色

【示例数据】
- 待办1：完成，标题"完成 React 项目"，分类 学习，优先级 高，截止 2025-03-25
- 待办2：未完成，标题"复习 TypeScript"，描述"重点看泛型"，分类 学习，优先级 中，截止 2025-03-28
- 待办3：未完成，标题"买 groceries"，分类 生活，优先级 低，无截止日期
```

## 二、分模块 Prompt（按需生成单个组件）

### 2.1 顶部统计面板

```
设计一个统计面板组件，横向排列：
- 左侧：完成率环形或条形进度，显示 "65% 完成"
- 右侧：待办数量卡片，显示 "7 项待办"
风格简洁，使用单一主色和灰色，无渐变。
```

### 2.2 添加待办表单

```
设计一个添加待办的表单：
- 标题输入框（必填）
- 描述多行文本框（选填）
- 分类下拉：工作、学习、生活
- 优先级下拉：低、中、高
- 截止日期选择器（日期输入）
- 主按钮："添加待办"
布局紧凑，label 清晰，风格简洁现代。
```

### 2.3 筛选栏

```
设计一个筛选栏，横向排列：
- 分类筛选下拉（全部、工作、学习、生活）
- 优先级筛选下拉（全部、低、中、高）
- 状态筛选：三个选项 全部 | 未完成 | 已完成（可用 tab 或 segment control）
风格与表单一致，紧凑易点。
```

### 2.4 待办卡片 (TodoItem)

```
设计一个待办事项卡片：
- 左侧：圆形勾选框（未完成/已完成两种状态）
- 中间：标题（粗体）、描述（灰色小字）、底部一行显示 分类标签 + 优先级标签 + 截止日期
- 右侧：编辑图标按钮、删除图标按钮
- 已完成状态：标题加删除线，整体略微变灰
卡片有轻微边框或背景区分，圆角 8-12px。
```

### 2.5 搜索框

```
设计一个搜索输入框：
- 占位符："搜索标题或描述..."
- 左侧有搜索图标
- 简洁风格，与表单输入框一致
```

## 三、移动端版本 Prompt

```
设计一个待办事项应用的移动端界面，375×812 画布（iPhone 尺寸）。

【布局】
- 顶部：标题 "我的待办"
- 统计：一行显示 "65% 完成 · 7 项待办"
- 添加按钮：悬浮或固定，点击展开添加表单
- 搜索框：全宽
- 筛选：可折叠或横向滚动的筛选 chips
- 待办列表：卡片纵向堆叠，每张卡片触控友好（至少 48px 高）
- 卡片内容：勾选框、标题、描述、分类/优先级/日期、编辑删除按钮

【风格】
- 简洁，单主色+灰色
- 触控区域充足
- 避免复杂装饰
```

## 四、设计规范摘要（供 Figma 变量/样式设置参考）

| 元素 | 建议值 |
|------|--------|
| 主色 | #2563eb（蓝）或 #059669（绿） |
| 背景 | #f8fafc 或 #f1f5f9 |
| 文字主色 | #1e293b |
| 文字次要 | #64748b |
| 边框 | #e2e8f0 |
| 圆角 | 8px 或 12px |
| 间距 | 8、16、24px 递进 |
| 优先级高 | 红色/橙色调 |
| 优先级中 | 黄色/琥珀色 |
| 优先级低 | 灰色/绿色 |

---

## 五、使用建议

1. **Figma AI / Generate UI**：复制「一、主界面完整 Prompt」整段，一次性生成主界面。
2. **组件拆分**：若生成结果不理想，可改用「二、分模块 Prompt」逐个生成，再组合。
3. **多尺寸**：先做桌面端，再复制「三、移动端版本」生成移动端，或手动适配。
4. **变量**：在 Figma 中创建 Color 和 Spacing 变量，与「四、设计规范摘要」对应，便于后续开发。
