## 一、项目概述

### 1.1 项目简介

本项目是一款轻量级待办事项管理应用，面向个人日常任务管理场景。用户可以通过该应用记录待办事项，设置优先级与截止日期，按分类组织任务，并通过筛选、搜索快速定位任务。数据持久化于浏览器 LocalStorage，无需后端即可使用。

### 1.2 目标用户

- 大一计算机学生（用于学习全栈开发）
- 需要简单任务管理的个人用户

### 1.3 核心价值

- **学习友好**：项目结构清晰，按模块循序渐进开发，适合作为学习案例
- **功能完整**：涵盖增删改查、筛选、搜索、统计等常见功能
- **技术现代**：使用 React + TypeScript + Vite 等主流技术栈

## 三、数据模型

### 3.1 待办事项 (Todo)

```typescript
interface Todo {
  id: string;              // 唯一标识，建议使用 crypto.randomUUID() 或 nanoid
  title: string;           // 标题，必填
  description?: string;    // 描述，选填
  category: string;        // 分类，如 "工作"、"学习"、"生活"
  priority: Priority;      // 优先级
  dueDate: string | null;  // 截止日期，ISO 8601 格式，可选
  completed: boolean;      // 是否已完成
  createdAt: string;       // 创建时间，ISO 8601 格式
}
```

### 3.2 优先级枚举

```typescript
type Priority = 'low' | 'medium' | 'high';
// 对应显示：低、中、高
```

### 3.3 筛选条件

```typescript
interface FilterState {
  category: string | null;   // null 表示全部
  priority: Priority | null;
  status: 'all' | 'active' | 'completed';
  searchQuery: string;
}
```

## 四、技术选型

| 类别 | 技术 | 说明 |
|------|------|------|
| 框架 | React 18+ | 组件化 UI 框架 |
| 语言 | TypeScript | 类型安全，提升可维护性 |
| 构建工具 | Vite | 快速开发服务器与构建 |
| 状态管理 | Zustand | 轻量级，比 Redux 简单，适合中小型项目 |
| 样式 | Tailwind CSS | 原子化 CSS，快速构建界面 |
| 日期处理 | date-fns | 轻量、不可变、Tree-shakeable |
| 数据存储 | LocalStorage | 浏览器本地存储，无需后端 |

## 五、开发顺序建议

建议按以下顺序实现，每完成一个模块即测试，确保稳定后再进入下一阶段。

| 阶段 | 模块 | 说明 |
|------|------|------|
| 1 | 数据存储层 | 封装 LocalStorage 读写，定义存取 key、序列化/反序列化 |
| 2 | 状态管理 | 用 Zustand 创建 store，定义 todos 与 filters 的 state 与 actions |
| 3 | 添加功能 | 实现添加待办表单与提交逻辑 |
| 4 | 列表展示 | 展示待办列表，支持完成/未完成切换 |
| 5 | 筛选功能 | 实现分类、优先级、状态筛选 |
| 6 | 编辑与删除 | 实现编辑弹窗与删除确认 |
| 7 | 搜索功能 | 实现关键词搜索 |
| 8 | 统计功能 | 实现完成率、待办数量等统计展示 |

## 六、非功能需求

### 6.1 兼容性

- 支持主流现代浏览器（Chrome、Firefox、Safari、Edge 最新两个版本）
- 响应式布局，适配桌面端与移动端

### 6.2 性能

- 首屏加载时间 < 3 秒（常规网络）
- 列表渲染支持虚拟列表（若待办数量 > 100）

### 6.3 可访问性

- 关键交互支持键盘操作
- 表单具备合理的 label 与错误提示
